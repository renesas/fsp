Detailed Description

Structure describing an area on a flash device.

Provides abstraction of flash regions for type of use. I.e. dude where's my image?

System will contain a map which contains flash areas. Every region will contain flash identifier, offset within flash and length.

  1. This system map could be in a file within filesystem (Initializer must know/figure out where the filesystem is at).
  2. Map could be at fixed location for project (compiled to code)
  3. Map could be at specific place in flash (put in place at mfg time).

Note that the map you use must be valid for BSP it's for, match the linker scripts when platform executes from flash, and match the target offset specified in download script. Multiple flash devices may be available in the system, each of which may have its own areas. For this reason, flash areas track which flash device they are part of.
