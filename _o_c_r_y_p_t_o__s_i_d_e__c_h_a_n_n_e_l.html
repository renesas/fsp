Ocrypto - Side Channel Safety Statement

Side-Channel Safety Statement

Copyright Oberon microsystems AG (since 2019)

1) Timing attacks, simple power/radiation analysis attacks

These attacks rely on data-dependent timing variations in the used algorithms. They can be applied passively, just by observing normal operation.

All ocrypto functions (see chapter 5) are PC-secure (execution paths do not depend on input data) and constant-time (execution time is independent of input data) except for the list of functions with special behavior listed below (see chapter 4).

A special case of timing variations are table reads with data-dependent indices. Accessing data-dependent memory addresses may influence the execution timing because of address-specific cache reload operations. Data-dependent table lookups may be dangerous even on systems without data cache because the table lookups in RAM or flash memory may dominate chip radiation if other calculations are done mainly in registers.

All ocrypto functions are free of data-dependent table reads except for the functions with special behavior listed below (see chapter 4).

2) Differential power/radiation analysis attacks

These attacks improve the precision of the analysis by averaging over several (typically hundreds of) runs and/or comparing several runs by feeding specific input data. They can only be applied if the system permits an arbitrary number of tries with either the same or arbitrarily chosen input data. Whether and what kind of countermeasures are needed for avoiding such attacks largely depends on the usage of the functions, not the functions themselves.

ocrypto does not contain any special provisions against differential attacks.

3) Fault attacks

In addition to differential analysis, fault attacks rely on actively influencing the attacked system to provoke faults in the execution. Countermeasures include physical protection of the device and/or attack detection mechanisms on all levels. Detection on an algorithm level is usually expensive and only justified if fault attacks are actually feasible for a given system.

ocrypto does not contain any special provisions against fault attacks.

4) List of ocrypto functions with special behavior

This list shows functions with special behavior:

  • Ed25519/Ed25519ph/Ed448/Ed448ph verify

    Ed25519 et.al. verify is not constant-time "by design". The algorithm is chosen such that a highly efficient variable-time double multiplication algorithm can be used. This is justified by the fact that signature verification depends on public values only (signed data, public signature key). The variable-time multiplication does not harm even if the data or the public key are considered to be secret, because neither of them is directly involved in the multiplication, only values derived from them using secure hash functions.

  • Ed25519/Ed25519/Ed448/Ed448ph public key and sign with "#define OCRYPTO_NO_CACHE"

    The point multiplication uses a table of base point multiples. Without caches, this poses no problems, but see the initial remark on table-based algorithms (chapter 1).

  • RSA with "#define OCRYPTO_NO_CACHE"

    The RSA exponentiation uses a table of small exponents of the argument. Without caches, this may pose no problems, but see the initial remark on table-based algorithms (chapter 1).

  • SRP

    The SRP calculations are not perfectly constant-time, because some hash values are taken on a compact representation of some critical values. In the worst case, this can reveal the number of leading zero bytes in these values, including the premaster secret. This cannot be avoided and is considered a design error in the SRP specification.

  • ML-DSA and ML-KEM

    ML-DSA and ML-KEM are not constant-time because of rejection sampling in several places. However, the timing can only disclose information on values that are not used.

    ML-DSA is not table-free because the SampleInBall algorithm permutes array elements with pseudo-randomly derived indices. However, the seed of the resulting polynomial is part of the signature and thus public anyway.
