Detailed Description

Ocrypto software only crypto on RA2E1.

Overview

Ocrypto is designed as a small footprint software crypto implementation to be used on resource constrained devices. The software only module is available in FSP only on RA2E1.

Note
The Ocrypto port module does not provide any crypto interfaces to the user. See Ocrypto Licensed Functions.

Hardware Overview

Limited to RA2E1 only.

Features

For features supported by the software-only version, refer to the Ocrypto documentation.

Configuration

Build Time Configurations for rm_ocrypto_trng_port

The following build time configurations are defined in fsp_cfg/rm_ocrypto_port_cfg.h:

ConfigurationOptionsDefaultDescription
Parameter Checking
  • Default (BSP)
  • Enabled
  • Disabled
Default (BSP) If selected code for parameter checking is included in the build.

AES Configuration

To enable hardware acceleration for the AES-EBC, CBC, and CTR operations, choose Ocrypto (H/W Accelerated) from the stack options. Be sure to initialize the hardware

Usage Notes

Hardware Initialization

Invoke RM_OCRYPTO_PORT_Init() to initialize the hardware before using Ocrypto if the TRNG or if H/W Acceleration is to be used.

Random Number Generation

Use the RM_OCRYPTO_PORT_TRNG_Read() function to obtain random data from the TRNG hardware. If purely random data is sufficient for the application, then RM_OCRYPTO_PORT_TRNG_Read() can be used directly instead.

Memory Usage

Ocrypto does not use dynamic allocation so there is no heap requirement.

A minimum stack of 0x800 is required to use Ocrypto.

Side Channel Safety Statement

See Ocrypto Side Channel Safety Statement.

Function Overview

Limitations

Usage with RA2E1

Ocrypto can only be used on RA2E1.

Ocrypto

Ocrypto does provide any API to generate keys. It is the responsibility of the caller to provide generated keys.

AES Reentrancy

Use the S/W only stack if AES reentrancy is required in your application.

Examples

Ocrypto Library Examples

For examples of how to use Ocrypto APIs, see Ocrypto test code under ra/oberon/ocrypto/tests/. These tests are not meant to be built or run in FSP and have been compiled out by the macro OCRYPTO_REFERENCE_TEST_CODE_DOES_NOT_BUILD by default.

TRNG Example

This is an example on using Ocrypto to generate a random number

#define TEST_SECRET_KEY_SIZE 32U
#define TEST_PUBLIC_KEY_SIZE 64U
void ocrypto_trng_example (void)
{
fsp_err_t status = FSP_SUCCESS;
uint8_t ecdsa_msg[3] = "abc";
uint8_t secret_key[TEST_SECRET_KEY_SIZE];
uint8_t session_key[TEST_SECRET_KEY_SIZE];
uint8_t ecdsa_public_key[TEST_PUBLIC_KEY_SIZE];
uint8_t signature[TEST_PUBLIC_KEY_SIZE];
/* Setup the platform; initialize the crypto engine. */
status = RM_OCRYPTO_PORT_Init();
if (FSP_SUCCESS != status)
{
assert(0);
}
/* Read random data from the TRNG to use as secret key. */
status = RM_OCRYPTO_PORT_TRNG_Read(secret_key, sizeof(secret_key));
if (FSP_SUCCESS != status)
{
assert(0);
}
if (0 != ocrypto_ecdsa_p256_public_key(ecdsa_public_key, secret_key))
{
assert(0);
}
/* Read random data from the TRNG to use as session key */
status = RM_OCRYPTO_PORT_TRNG_Read(session_key, sizeof(session_key));
if (FSP_SUCCESS != status)
{
assert(0);
}
if (0 != ocrypto_ecdsa_p256_sign(signature, ecdsa_msg, sizeof(ecdsa_msg), secret_key, session_key))
{
assert(0);
}
if (0 != ocrypto_ecdsa_p256_verify(signature, ecdsa_msg, sizeof(ecdsa_msg), ecdsa_public_key))
{
assert(0);
}
/* Operation successful. */
while (1)
{
;
}
}
