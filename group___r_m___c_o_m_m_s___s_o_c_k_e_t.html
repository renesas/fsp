Detailed Description

Middleware to implement a generic communications interface over LWIP. This module implements the Communications Middleware Interface.

Overview

The RM_COMMS_SOCKET module implements COMMS API for LWIP interface.

Features

The implementation of the LWIP communications interfacehas the following key features:

  • Non-blocking API for bare metal
  • Non-blocking and blocking API for RTOS
  • Socket connection detection mode

Usage Notes

Limitations

  • If DHCP is enabled, user have to find board's IP address via DHCP server, mDNS, etc.
  • UDP socket is not supported.
  • RM_COMMS_SOCKET_WriteRead API is not implemented.

Examples

Basic Example

This is a basic example of minimal use of LWIP communications implementation in an application.

void rm_comms_socket_example () {
/* This step: Initialize pin setting (of MCU device) */
/* TODO code */
rm_comms_socket_extended_cfg_t * p_extend = (rm_comms_socket_extended_cfg_t *) g_comms_socket_port_cfg.p_extend;
struct netif comms_socket_netif;
/* Following code are lwIP setup process. */
lwip_init();
/* Initialize netif before open rm_comms_socket. */
netif_add(&comms_socket_netif,
RM_COMMS_SOCKET_EXAMPLE_IP_ADDR,
RM_COMMS_SOCKET_EXAMPLE_NETMASK,
RM_COMMS_SOCKET_EXAMPLE_GW,
p_extend->p_lwip_ether,
rm_lwip_ether_init,
netif_input);
netif_set_default(&comms_socket_netif);
netif_set_up(&comms_socket_netif);
netif_set_link_up(&comms_socket_netif);
/* Init the rm_comms_socket */
RM_COMMS_SOCKET_Open(&g_comms_socket_port_ctrl, &g_comms_socket_port_cfg);
/* Waiting few milliseconds for socket ready to be used */
R_BSP_SoftwareDelay(10, BSP_DELAY_UNITS_MILLISECONDS);
/* Set callback function */
RM_COMMS_SOCKET_CallbackSet(&g_comms_socket_port_ctrl, tcp_callback, g_comms_socket_port_cfg.p_context);
/* This step: Client start to creating the socket and send connect command to Server (MCU device) */
/* TODO code */
/* Waiting for connection to be established. */
LWIP_TEST_WAIT_TIMEOUT(true == connected, LWIP_TEST_MAXIMUM_WAITING_TIME);
/* Tries 3 times loop to receive/send data from/to client */
for (uint16_t tries = 1; tries < 3; tries++)
{
/* This step: Server (MCU device) starts to receive data */
RM_COMMS_SOCKET_Read(&g_comms_socket_port_ctrl, rx_tcp_data, BUFFER_SIZE);
/* This step: Client starts to send the data to Server (MCU device) */
/* TODO code */
/* Wait for receiving data, RX complete flag will be indicated in callback function */
LWIP_TEST_WAIT_TIMEOUT(true == rx_complete, LWIP_TEST_MAXIMUM_WAITING_TIME);
/* Reset flag */
rx_complete = false;
/* Server (MCU device) starts sending package */
RM_COMMS_SOCKET_Write(&g_comms_socket_port_ctrl, tx_tcp_data, BUFFER_SIZE);
/* Wait for TX completed */
LWIP_TEST_WAIT_TIMEOUT(true == tx_complete, LWIP_TEST_MAXIMUM_WAITING_TIME);
/* This step: Client starts to receive data from server (MCU device) */
/* TODO code */
/* Clear the TX flag */
tx_complete = false;
}
/* Close the socket server */
RM_COMMS_SOCKET_Close(&g_comms_socket_port_ctrl);
}
void tcp_callback (rm_comms_callback_args_t * p_args)
{
switch (p_args->event)
{
case RM_COMMS_EVENT_ERROR:
{
err = true;
break;
}
case RM_COMMS_EVENT_RX_OPERATION_COMPLETE:
{
rx_complete = true;
break;
}
case RM_COMMS_EVENT_TX_OPERATION_COMPLETE:
{
tx_complete = true;
break;
}
case RM_COMMS_EVENT_OPERATION_COMPLETE:
{
connected = true;
break;
}
}
}
